# 魔法陣タイピングバトル — 要件定義書 v2.0

## 1. プロジェクト概要

### 1.1 コンセプト

タイピングによって魔法陣を完成させ、呪文（破道・縛道）を発動して敵を倒すアクション寄りのタイピングゲーム。
呪文はBLEACHの鬼道をオマージュした日本語詠唱で表示され、プレイヤーはローマ字入力で詠唱を行う。
複雑で長い呪文ほど強力な魔法が発動し、大ダメージを与えることができる。
プレイヤーは攻撃だけでなく防御呪文も駆使し、敵の攻撃をしのぎながら戦う。

### 1.2 ジャンル

タイピングアクション × サバイバル

### 1.3 対象プラットフォーム

- Webブラウザ（PC）
- キーボード入力前提（タッチデバイス非対応）

### 1.4 技術スタック

| 項目 | 技術 | 選定理由 |
|------|------|---------|
| フロントエンド | React + TypeScript | UI管理とコンポーネント設計 |
| 魔法陣描画 | HTML Canvas / WebGL | SVGではリッチなアニメーション時にDOM再描画コストが高くなるため、Canvas系を採用。60fps維持が必須 |
| UIレイヤー | React（DOM） | HUD・テキスト表示などはReact管理。描画エンジンとUI層を分離する |
| アニメーション | requestAnimationFrame + Canvas API | フレーム制御を自前で管理し、パフォーマンスを確保 |
| ビルド | Vite | 高速なHMRと軽量バンドル |
| ホスティング | 静的サイト（Vercel / Netlify 等） | — |

## 2. デザインの方向性 — BLEACHの美学

### 2.1 コアとなるデザイン哲学

久保帯人の作風を参考に、以下の3原則をゲーム全体のビジュアル設計に適用する。

**① 黒と白のコントラスト**
久保帯人は「主人公のイメージカラーを黒に設定し、白を効果的に使うことでコントラストを意識するようになった」と語っている。本ゲームでも、黒を基調とした画面に白い光の演出を重ねる構成を基本とする。

**② 余白（ネガティブスペース）の活用**
「何も描かれていない余白の部分こそが大事。何もないことが"溜め"になって次のコマに生きる」（久保帯人, 2008年）。画面を要素で埋め尽くさず、余白を意図的に残すことで、魔法陣の完成や呪文発動の瞬間にインパクトを持たせる。

**③ シャープで洗練されたライン**
千年血戦篇で顕著な、鋭角的で洗練されたデザインライン。魔法陣の幾何学模様もシャープな直線・鋭角を多用し、装飾過多にならないミニマルなデザインを目指す。

### 2.2 カラースキーム

| 要素 | カラー | 備考 |
|------|--------|------|
| 背景 | 漆黒（#0A0A0A〜#111111） | 完全な黒ではなく、わずかに色味を持たせる |
| 魔法陣（通常） | 淡い青白（#A0C4FF → #E0EEFF） | 霊圧をイメージした冷たい光 |
| 魔法陣（コンボ時） | 後述のコンボカラーテーブル参照 | 段階的に暗く、最終的に黒+白縁へ |
| UIテキスト | 白（#FFFFFF）+ グレー（#888888） | 重要度で濃淡を分ける |
| ダメージ表示 | 白（プレイヤー攻撃） / 赤（被ダメージ） | — |
| 制限時間バー | 白 → 赤（残り少ない時） | — |
| 敵HP | 暗い赤（#8B0000） | — |

### 2.3 タイポグラフィ

| 用途 | フォント系統 | 備考 |
|------|-------------|------|
| 呪文テキスト（日本語） | 明朝体系（Noto Serif JP 等） | 詠唱の荘厳さを表現 |
| ローマ字ガイド | 等幅フォント（Fira Code 等） | タイピング判定のため1文字ずつ明確に |
| UI（スコア・HP等） | ゴシック体系（Noto Sans JP 等） | 可読性重視 |
| ウェーブ表示・ボス名 | 極太ゴシック or カスタム | BLEACHのチャプタータイトル風 |

### 2.4 画面演出の原則

- 画面の揺れ（スクリーンシェイク）は控えめに。余白と静寂で「溜め」を作り、発動の瞬間だけ動きを入れる
- パーティクルは白い光の粒子を基本とし、属性カラーを控えめに添える程度
- 敵のシルエットは黒を基調とし、目や輪郭だけが光るような表現

## 3. ゲームフロー

### 3.1 全体フロー

```
タイトル画面
    ↓
チュートリアル（呪文習得フェーズ）
    ├─ 攻撃呪文を1つずつ練習（5種）
    ├─ 防御呪文を練習（1〜2種）
    └─ 模擬戦（弱い敵で総合練習）
    ↓
本編（エンドレスサバイバル）
    ├─ ウェーブ 1: 通常フェーズ（雑魚敵）
    ├─ ウェーブ 1: ボスフェーズ
    ├─ ウェーブ 2: 通常フェーズ（雑魚敵）
    ├─ ウェーブ 2: ボスフェーズ
    ├─ ... 繰り返し（難易度漸増）
    └─ ゲームオーバー
    ↓
リザルト画面
```

### 3.2 チュートリアル（呪文習得フェーズ）

ゲーム本編に入る前に、全ての呪文を実際にタイプして覚えるフェーズを設ける。

| ステップ | 内容 | 制限時間 |
|----------|------|---------|
| 1 | 低級攻撃呪文を2つ練習。各呪文を1回ずつ成功させる | なし（自由練習） |
| 2 | 中級攻撃呪文を2つ練習 | なし |
| 3 | 高級攻撃呪文を1つ練習 | なし |
| 4 | 防御呪文を練習 | なし |
| 5 | 模擬戦。弱い敵1体を相手に、呪文を自由に選んで戦う。敵の攻撃も発生する | ゆるめ |

- チュートリアルはスキップ可能にする（2回目以降のプレイのため）
- 各呪文の練習中は、呪文の名前・属性・威力が表示される

### 3.3 ゲームモード：エンドレスサバイバル

#### 通常フェーズ

- 雑魚敵が1体ずつ順番に出現する
- 画面に表示された呪文をタイプして敵を倒す（呪文はシステムが指定）
- 各敵に制限時間あり。制限時間を超えると敵が攻撃してプレイヤーにダメージ

#### ボスフェーズ

- ボス敵が1体出現。HPが高く、複数回の呪文が必要
- **画面の隙間に5種の呪文が常時表示**される
- プレイヤーは**好きな呪文を好きなタイミングで好きなだけ**選んで発動できる
- 短い呪文で安全に削るか、長い呪文で大ダメージを狙うかの戦略的判断が生まれる
- ボスは定期的に攻撃を仕掛けてくる。攻撃を受ける前に防御呪文で防ぐことが重要

#### ウェーブ進行

| ウェーブ | 雑魚数 | ボスHP目安 | 制限時間 | 敵の攻撃頻度 |
|----------|--------|-----------|---------|-------------|
| 1 | 3体 | 低 | 余裕あり | 低い |
| 2〜3 | 4体 | 中 | やや短い | 普通 |
| 4〜6 | 5体 | 高 | 短い | やや高い |
| 7以降 | 5〜6体 | 非常に高 | かなり短い | 高い |

- ウェーブ間に短いインターバル（2〜3秒）。ウェーブ番号を表示

## 4. プレイヤーシステム

### 4.1 プレイヤーHP

プレイヤーにHP（体力）を導入する。HPが0になった時点でゲームオーバー。

| パラメータ | 値 |
|------------|-----|
| 最大HP | 100 |
| 回復手段 | なし（サバイバルのため） |

### 4.2 被ダメージ要因

| 要因 | ダメージ量 | 発生タイミング |
|------|-----------|---------------|
| 通常フェーズ：制限時間超過 | 中ダメージ（20〜30） | 制限時間内に敵を倒せなかった場合 |
| ボスフェーズ：ボスの攻撃 | 中ダメージ（15〜25） | ボスが定期的に攻撃を仕掛ける |
| ボスフェーズ：制限時間超過 | 大ダメージ（30〜40） | ボス戦全体の制限時間切れ |

### 4.3 被ダメージ演出

- 画面の縁が赤く一瞬フラッシュ
- HPバーが減少（アニメーション付き）
- ダメージ数値が白テキストで浮かび上がる

### 4.4 防御呪文

敵の攻撃を防ぐための縛道（防御呪文）。ボス戦で攻撃の予兆が表示されたら、素早く防御呪文をタイプすることでダメージを無効化またはカットする。

| 種別 | 判定 |
|------|------|
| 完全防御 | 攻撃の予兆中に防御呪文を完全にタイプ → ダメージ0 |
| 不完全防御 | 途中までタイプした状態で攻撃を受ける → ダメージ半減 |
| 防御失敗 | 防御呪文をタイプしていない → フルダメージ |

#### 攻撃の予兆

- ボスが攻撃する2〜3秒前に、警告表示（画面上に「⚠ 攻撃が来る！」的な演出）
- この間にプレイヤーは攻撃呪文の入力を中断して防御呪文に切り替えるか、攻撃を続行するかを判断する
- 攻撃呪文の入力途中で防御に切り替えた場合、攻撃呪文の進捗はリセットされる

## 5. タイピングシステム

### 5.1 入力方式

- **ローマ字入力**
- 呪文テキストは「日本語表示 + その下にローマ字表示」の2段構成
- 入力はテキストフィールドではなく、キーボードイベント（keydown）を直接取得する方式

### 5.2 ローマ字判定エンジン（状態遷移マシン）

ローマ字の揺れ対応は独自実装の泥沼を避けるため、**状態遷移マシン（ステートマシン）として設計**する。

#### 設計方針

1. 呪文テキスト（ひらがな）を**文節単位**に分割する
2. 各ひらがな文字について、許容されるローマ字入力パターンを全てグラフとして定義する
3. プレイヤーのキー入力に対して、現在の状態から遷移可能かをリアルタイムに判定する

#### 主要なローマ字揺れ対応

| ひらがな | 許容パターン |
|----------|-------------|
| し | si, shi |
| ち | ti, chi |
| つ | tu, tsu |
| ふ | hu, fu |
| じ | zi, ji |
| しゃ | sya, sha, sixya |
| ちゃ | tya, cha, tixya |
| っ（促音） | 次の子音を2回 / xtu / ltu |
| ん | nn / n（次が母音・na行・ya行以外なら n 1回で可） |

#### 「ん」の特殊処理

「ん」の判定は最も複雑であり、次の文字によって挙動が変わる。

| 次の文字 | 「ん」の入力 |
|----------|-------------|
| 子音で始まる文字（か・さ・た行等） | n 1回で確定 |
| 母音・な行・や行 | nn が必須（n 1回では次の文字の入力と区別できないため） |
| 文末 | nn が必須 |

この処理は状態遷移マシンの中で、次の入力文字を先読みして分岐させる。

### 5.3 入力判定とフィードバック

| 入力 | フィードバック |
|------|--------------|
| 正しい入力 | 該当文字がハイライト（白→明るい光）。次の文字へ進む |
| 誤入力 | ミス音 + 画面の微振動。**文節の先頭まで巻き戻る** |

### 5.4 ミスタイプのペナルティ：文節巻き戻し

ミスタイプが発生した場合、**現在タイプ中の文節（一文）の先頭まで進捗が巻き戻る**。

#### 具体例

呪文：「裂けよ、烈火」（さけよ、れっか）

```
入力中：s-a-k-e-y-o-r-e-k [ここでミス]
→「れっか」の先頭まで戻る
→「裂けよ、」は確定済みのため維持
→「烈火」部分を再入力
```

#### 文節の定義

- 読点（、）区切り、または助詞の切れ目で文節を分割する
- 呪文データ側で文節区切り位置を明示的に定義する（後述の呪文データ構造参照）

#### 魔法陣への影響

- 文節巻き戻しに連動して、**魔法陣の描画進捗も該当分だけ巻き戻る**
- 完成しかけた魔法陣のレイヤーが消えていく演出（逆再生風）

## 6. 呪文システム

### 6.1 呪文の分類

BLEACHの鬼道に倣い、攻撃呪文（破道）と防御呪文（縛道）の2系統を設ける。

#### 攻撃呪文（破道）— 5種

| # | 呪文名 | ランク | 詠唱テキスト例 | 文字数目安 | 基礎ダメージ |
|---|--------|--------|---------------|-----------|-------------|
| 1 | 衝（さい） | 低級 | 「裂けよ」 | 3〜5文字 | 15 |
| 2 | 赤火砲（しゃっかほう） | 低級 | 「君臨者よ、血肉の仮面」 | 10〜15文字 | 30 |
| 3 | 蒼火墜（そうかつい） | 中級 | 「君臨者よ、血肉の仮面、万象、羽搏き、ヒトの名を冠す者よ、真理と節制、罪知らぬ夢の壁に僅かに爪を立てよ」 | 30〜40文字 | 60 |
| 4 | 雷吼炮（らいこうほう） | 中級 | 「散在する獣の骨、尖塔、紅晶、鋼鉄の車輪、動けば風、止まれば空、槍打つ音色が虚城に満ちる」 | 30〜40文字 | 70 |
| 5 | 黒棺（くろひつぎ） | 高級 | 「滲み出す混濁の紋章、不遜なる狂気の器、湧きあがり否定し、痺れ瞬き、眠りを妨げる、爬行する鉄の王女、絶えず自壊する泥の人形、結合せよ、反発せよ、地に満ちよ己の無力を知れ」 | 60文字以上 | 150 |

#### 防御呪文（縛道）— 2種

| # | 呪文名 | 詠唱テキスト例 | 文字数目安 | 効果 |
|---|--------|---------------|-----------|------|
| 1 | 斥（せき） | 「退けよ」 | 3文字 | 完全防御（短いが判定がシビア） |
| 2 | 断空（だんくう） | 「拒絶する白壁、禁じられし境界」 | 15文字 | 完全防御 + 次の攻撃のダメージ1.5倍バフ |

### 6.2 呪文の出現ルール

| フェーズ | 呪文の出方 |
|----------|-----------|
| 通常フェーズ | システムが呪文を指定。低級〜中級がランダム出現。ウェーブが進むと中級以上の出現率が上がる |
| ボスフェーズ | **画面に5種の攻撃呪文 + 防御呪文が常時表示**。プレイヤーが自由に選択して入力 |

#### ボスフェーズの呪文選択UI

- 画面左端または右端に、呪文リストを縦に配置する
- 各呪文に対応するショートカットキー（1〜5 + Q/W）を設定
- キーを押すと呪文が選択され、即座にローマ字入力モードに入る
- 入力途中で別の呪文キーを押すと、現在の入力をキャンセルして切り替え可能

### 6.3 呪文データ構造

```json
{
  "id": "hado_033",
  "name": "蒼火墜",
  "name_reading": "そうかつい",
  "type": "attack",
  "rank": "mid",
  "text_jp": "君臨者よ、血肉の仮面、万象、羽搏き、ヒトの名を冠す者よ、真理と節制、罪知らぬ夢の壁に僅かに爪を立てよ",
  "segments": [
    "くんりんしゃよ",
    "けつにくのかめん",
    "ばんしょう",
    "はばたき",
    "ひとのなをかんすものよ",
    "しんりとせっせい",
    "つみしらぬゆめのかべにわずかにつめをたてよ"
  ],
  "base_damage": 60,
  "element": "fire",
  "magic_circle_type": "hexagram"
}
```

- `segments`：文節ごとの区切り。ミスタイプ時の巻き戻し単位として使用する
- `element`：魔法陣のカラーリングに影響する属性
- `magic_circle_type`：魔法陣の形状パターン

## 7. 魔法陣システム（演出重視）

### 7.1 基本コンセプト

タイピングの進捗に応じて魔法陣がリアルタイムで描画・完成していく。
BLEACHの美学を反映し、黒い背景に白〜青白い光のラインで魔法陣を描く。
コンボが溜まるにつれ魔法陣の色が暗く変化し、最大コンボでは黒い魔法陣に白い縁取りだけが浮かぶ。

### 7.2 魔法陣の構成要素と描画段階

Canvas上で以下のレイヤーを段階的に描画する。

| 進捗 | 描画レイヤー | 演出 |
|------|--------------------|------|
| 0〜20% | 外周の円 | 光の線が走るように円が描かれる |
| 20〜40% | 内側の幾何学模様（六芒星・五芒星等） | 頂点から頂点へ線が繋がっていく |
| 40〜60% | 呪文テキスト（円周上に配置） | 1文字ずつ浮かび上がる |
| 60〜80% | 内側の紋様・中央のシンボル | 脈打つように光る |
| 80〜100% | 全体完成 + 放射光 | 溜めの静寂 → 光が解放される |

### 7.3 コンボによる魔法陣のカラー変化

コンボが溜まるにつれ、魔法陣の色がBLEACH的な「黒」へ近づいていく。

| コンボ段階 | 魔法陣のカラー | 演出の変化 |
|-----------|---------------|-----------|
| 0〜4（通常） | 淡い青白（#A0C4FF） | 標準的な光の魔法陣 |
| 5〜9 | 深い青（#2E4A7A） | 光が引き締まり、線がシャープに |
| 10〜14 | 濃紫（#3A1A5E） | 魔法陣の内側に暗い影が滲む |
| 15〜19 | 暗い赤紫（#2A0A2A） | 不穏なオーラ。背景にも暗い波動 |
| 20以上 | **黒（#0A0A0A）+ 白い縁（#FFFFFF）** | 最大演出。黒い魔法陣の輪郭だけが白く光る。BLEACHの「黒棺」を彷彿とさせる禍々しい美しさ |

- 色の遷移はリニアではなく、段階的に切り替わる
- 切り替わりの瞬間に一瞬のフラッシュ演出を入れる

### 7.4 属性カラー（通常時のアクセント）

呪文の属性に応じて魔法陣のアクセントカラーが変化する。コンボカラーが優先だが、コンボ段階が低い時は属性色が見える。

| 属性 | アクセントカラー |
|------|-----------------|
| 火 | 赤みがかった白（#FFD4C4） |
| 雷 | 紫がかった白（#D4C4FF） |
| 闇 | 紫黒（#2A0A3A） |
| 光 | 純白（#FFFFFF） |
| 結界 | 淡い緑白（#C4FFD4） |

### 7.5 発動演出

呪文を完全にタイプし終えると、以下の発動演出が再生される。

| ステップ | 時間 | 演出内容 |
|---------|------|---------|
| 1. 溜め | 0.2秒 | 魔法陣の回転が停止。画面全体がわずかに暗転。**余白と静寂** |
| 2. 集束 | 0.3秒 | 中央に光（またはエネルギー）が収束 |
| 3. 発射 | 0.3秒 | エネルギーが敵に向かって放たれる |
| 4. 着弾 | 0.3秒 | 敵にヒット。ダメージ数値が浮かび上がる |
| 5. 消散 | 0.4秒 | 魔法陣がフェードアウト |

合計約1.5秒。テンポを崩さない範囲でリッチにする。
コンボ20以上の場合は溜めのステップを0.5秒に延長し、より重厚感を出す。

### 7.6 ミスタイプ時の巻き戻し演出

- 文節巻き戻しに連動して、魔法陣の該当レイヤーが逆再生のように消えていく
- 消滅時に小さな光の粒子が散る演出
- 全体がリセットされるわけではなく、進捗が後退した分だけ影響する

## 8. コンボシステム

### 8.1 コンボの定義

| 条件 | コンボへの影響 |
|------|---------------|
| 呪文を完了（ミスなし） | コンボ +1 |
| 呪文を完了（ミスあり） | コンボ維持（増加しない） |
| 被ダメージ（防御失敗） | コンボリセット（0に戻る） |

### 8.2 コンボによるダメージ倍率

| コンボ数 | ダメージ倍率 |
|----------|-------------|
| 0〜4 | ×1.0 |
| 5〜9 | ×1.5 |
| 10〜14 | ×2.0 |
| 15〜19 | ×2.5 |
| 20以上 | ×3.0 |

### 8.3 コンボの演出フィードバック

魔法陣のカラー変化（7.3参照）に加えて、以下の演出を段階的に追加する。

| コンボ段階 | 追加演出 |
|-----------|---------|
| 5以上 | コンボ数表示が大きくなる。画面の四隅にかすかなオーラ |
| 10以上 | 魔法陣の描画速度がやや速くなる（テンポアップ感） |
| 15以上 | 背景に暗い波動のパルスが周期的に走る |
| 20以上 | 黒+白の魔法陣。画面全体に静かな威圧感。コンボ数表示が白く光る |

## 9. 制限時間の設計

### 9.1 動的制限時間

プレイヤーのタイピング速度の個人差に対応するため、制限時間は呪文の文字数に基づいて動的に計算する。

```
制限時間（秒） = 呪文のローマ字文字数 ÷ 基準タイピング速度 × 難易度係数 + バッファ
```

| パラメータ | 値 |
|------------|-----|
| 基準タイピング速度 | 3.0 打/秒（初心者〜中級者を想定） |
| 難易度係数 | ウェーブ1: 2.0 → ウェーブ5: 1.2 → ウェーブ10以降: 0.8 |
| バッファ | 2.0秒（呪文切替等の猶予） |

例：ローマ字30文字の呪文 × ウェーブ1の場合
→ 30 ÷ 3.0 × 2.0 + 2.0 = **22.0秒**

### 9.2 ボスフェーズの制限時間

ボスフェーズは個別呪文の制限時間ではなく、**ボス戦全体の制限時間**を設ける。

| ウェーブ | ボス全体の制限時間 |
|----------|------------------|
| 1 | 60秒 |
| 2〜3 | 50秒 |
| 4〜6 | 45秒 |
| 7以降 | 40秒 |

時間切れの場合、ボスの大攻撃が発動し、大ダメージを受ける（即死ではない）。

### 9.3 制限時間の表示

- 画面下部にバー形式で表示
- 残り30%で色が白→赤に変化
- 残り10%で点滅 + 画面の縁が赤くパルスする

## 10. スコアシステム

### 10.1 スコア計算

```
1呪文のスコア = 基礎点 × コンボ倍率 × タイピング速度ボーナス × ノーミスボーナス
```

| 要素 | 計算方法 |
|------|---------|
| 基礎点 | 呪文の基礎ダメージ × 10（衝: 150 / 赤火砲: 300 / 蒼火墜: 600 / 雷吼炮: 700 / 黒棺: 1500） |
| コンボ倍率 | 8.2 参照 |
| 速度ボーナス | 制限時間の残り割合。80%以上残し: ×1.5 / 50%以上: ×1.2 / それ以下: ×1.0 |
| ノーミスボーナス | ミスタイプ0回で完了: ×1.5 / 1回以上: ×1.0 |

### 10.2 リザルト表示

ゲームオーバー時に以下を表示する。

| 項目 | 内容 |
|------|------|
| 最終スコア | 合計スコア |
| 到達ウェーブ | 何ウェーブまで到達したか |
| 最大コンボ | 最大コンボ数 |
| 撃破数 | 倒した敵の総数 |
| ノーミス回数 | ミスなしで完了した呪文の回数 |
| 総タイプ数 | 合計キータイプ数 |
| 正確率 | 正しいタイプ数 ÷ 総タイプ数 × 100 |

### 10.3 ハイスコア保存

- ローカルストレージにTop10のハイスコアを保存
- リザルト画面でハイスコア更新時は専用の演出を表示
